  • iOS 앱개발 숙련 강의 수강

느낀점

오늘 강의에서 Life Cycle(생명주기)을 배웠다

iOS 앱에서의 라이프사이클이란 만들어진 앱이 시작할떄부터 종료될때까지의 모든 이벤트를 의미하며 이 라이프사이클을 잘 이해해야만 앱의 동작과 상태변화에 따라 앱을 관리하고 적절한 대응을 할수가 있다고 한다

Not Running :
앱이 아직 실행되지 않은 상태. 앱은 동작하지 않는다

Inactive (Foreground) :
앱이 포그라운드, 실행 중에는 있지만 아직 이벤트는 받을 수 없는 상태
앱 실행 중에 멀티태스킹 화면 진입이나, 알림 전화 등일 경우 이 상태가 된다.

Active (Foreground) :
앱이 활성화된 상태.
사용자로부터 이벤트를 받아서 상호작용할 수 있다.

Running (Background) :
앱이 백그라운드에서 실행되고 있는 상태
예를 들어 아이폰에서 Youtube나 Spotify를 실행하다가 화면을 끄거나 다른 앱을 실행해도
백그라운드에서 계속 소리가 들리는 작업등이 이에 해당된다.

Suspended (Background) :
앱이 백그라운드로 전환된 상태에서 메모리 부족과 같은 시스템 리소스의 문제로 앱이 실행되지 않고 있는 상태
앱이 일시적으로 중단되므로 코드의 실행은 중단되어 상호작용이나 데이터 처리는 불가능하나
상태는 보존되어 있어 시스템 리소스가 충분하면 다시 포그라운드 상태로 돌아올 수 있다.
단! 메모리가 부족하면 Suspended 상태에 있는 앱들부터 메모리에서 해제시키므로 앱을 다시 실행했을 때 처음부터 다시 실행하게 되는 경우도 있다.

Terminated :
앱이 완전히 종료된 상태. 사용자나 시스템에 의해 강제 종료된 상태

또 뷰의 라이프사이클도 있는데

뷰 라이프사이클은 뷰 컨트롤러 내부의 뷰 객체가 생성되고 제거되는 과정을 말한다. 각 뷰 컨트롤러 당 하나 이상의 뷰를 관리하며 각 뷰마다 그 자체의 라이프사이클을 가진다.

예를 들어 뷰가 화면에 나타내기 직전에 viewWillAppear메서드가,
뷰가 화면에서 사라진 직후에 viewDidDisappear메서드가 호출되는 것이
위에서 앱의 라이프사이클이 앱의 시작부터 종료까지의 전체 과정이라면
뷰의 라이프사이클은 뷰 컨트롤러 내부의 뷰 객체가 생성되고 제거되는 과정을 말한다.
이 두 라이프사이클은 앱 내에서 서로 연관되어 있어서 앱이 활성화되면서 뷰의 라이프사이클도 변화하게 된다

viewDidLoad :
뷰 계층이 로드되면 호출됨. 초기화 및 설정 작업 수행

viewWillAppear :
뷰가 화면에 나타나기 직전에 호출. 애니메이션 효과 관련된 작업 수행

viewDidAppear :
뷰가 화면에 나타난 직후에 호출. 사용자와의 상호작용 준비

viewWillDisappear :
뷰가 화면에서 사라지기 직전에 호출, 애니메이션 효과와 관련된 작업 수행

viewDidDisappear :
뷰가 화면에서 사라진 직후에 호출, 정리작업이나 다른 화면으로의 전환 수행

아래는 예시 코드이다

import UIKit

class ViewController: UIViewController {

    override func viewDidLoad() {
        super.viewDidLoad()
        print("View Did Load")
      	// 초기화 작업이나 데이터 로드 등을 여기에 작성
    }

    override func viewWillAppear(_ animated: Bool) {
        super.viewWillAppear(animated)
        print("View Will Appear")
      	// 화면에 나타나기 전에 UI업데이트나 데이터 갱신 등을 처리하는 코드 여기에 작성
    }

    override func viewDidAppear(_ animated: Bool) {
        super.viewDidAppear(animated)
        print("View Did Appear")
      	// 화면이 나타난 후에 애니메이션이나 사용자 입력 감지 등 처리 코드 여기에 작성
    }

    override func viewWillDisappear(_ animated: Bool) {
        super.viewWillDisappear(animated)
        print("View Will Disappear")
      	// 화면에서 사라지기 전에 데이터 저장이나 설정 변경 등을 여기에 작성
    }

    override func viewDidDisappear(_ animated: Bool) {
        super.viewDidDisappear(animated)
        print("View Did Disappear")
      	// 화면에서 사라진 후에 정리 작업이나 리소스 해제 등 여기에 작성
    }

    deinit {
        print("Deinit")
      	// 뷰 컨트롤러가 메모리에서 해제되기 전에 필요한 정리 작업 여기에 작성
    }
}
